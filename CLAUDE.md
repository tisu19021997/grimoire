# grimoire

A personal vault for non-linear learning. Notes, clippings, chats, sketches, code — anything worth re-reading.

Inspired by Karpathy's LLM wiki approach: https://gist.github.com/karpathy/442a6bf555914893e9891c11519de94f — **compile knowledge once at ingest, query the compiled vault forever.**

## spirit

- **Messy on purpose.** No taxonomy upfront. Folders earn their existence.
- **Non-linear.** Ideas should collide. Cross-link generously. Diverge, converge, repeat.
- **Short but not dull.** Cut filler, keep voice. Bold the line worth stopping for.
- **Random is good.** Surface old notes next to new ones.
- **Learn my taste.** Pay attention to what I save, dwell on, return to. Refine `taste.md` over time.

## what lives here

Markdown, mostly. Code when it earns it. Sources from everywhere:

- Obsidian web clipper drops
- LLM/agent chats I want to keep
- Hand-typed notes
- Things you (Claude) source for me on a cadence

This will be opened as an Obsidian vault, so use `[[wikilinks]]` and `#tags`.

## structure (deliberately thin)

- `log.md` — append-only timeline: date, link, one line on why it matters
- `taste.md` — your evolving model of me; update when patterns shift
- `index.md` — light hub, just enough to not get lost
- everything else — flat, or folders only when a cluster earns one

Don't manufacture structure. Don't migrate things to be "tidy." Let it grow weird.

Hobbies, not a fence — surprise me with adjacent things.

## how to work in this vault

**When I drop a source (clip, chat, link, snippet):**
1. Read it. Extract the spark, not a bland summary.
2. File as `kebab-case-title.md`. Front-matter: `date`, `source`, `tags`, optional `mood`.
3. Cross-link aggressively. Reach for **surprising** connections over obvious ones.
4. Append one line to `log.md`.
5. Update `taste.md` if you noticed a shift.

**When I ask a question:**
- Search the vault first. Synthesize from my own notes when you can.
- If I learn something new in the answer, file it back. The vault compounds.

**When you source on a cadence:**
- Bias toward variety. Mix domains. Don't echo-chamber me.
- Lead with the hook, not the headline. One paragraph max per item.
- **Bold** the sentence that should make me stop scrolling.
- End with a question or a thread to pull, not a wrap-up.

**On highlighting:** I scan. Use **bold** for the one thing per note that matters most. Use callouts (`> [!note]`, `> [!idea]`) sparingly — they should mean something.

## anti-rules

- No grand reorganizations. No README. No "let's restructure."
- Don't gate-keep what's "worthy." If I saved it, it's in.
- Don't be exhaustive. Be interesting.
- Don't apologize for messiness. That's the point.
